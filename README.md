# ProMatchAI

ProMatchAI is a mobile-first web application that helps refugees and job seekers build employability profiles, discover relevant jobs and apprenticeships, track applications, and access learning resources. It also includes employer/admin workflows for managing opportunities, reviewing applicants, shortlisting candidates, and viewing platform analytics.

The project is built as a Vite + React application with a Supabase Edge Function backend and a Capacitor Android wrapper.

## Features

- Multilingual onboarding for refugees, job seekers, students, professionals, and employers
- Profile creation with skills, education, experience, languages, interests, and work preferences
- AI-style job and candidate matching based on skills, profile completeness, interests, and opportunity data
- Job browsing, applications, application status tracking, and notifications
- Learning tab with course progress, certificates, and profile achievement signals
- Employer/admin panel for job, course, applicant, shortlist, and application management
- Mobile-focused UI with a dedicated mobile demo mode
- Supabase Edge Function API for users, jobs, courses, applications, analytics, and recommendations
- Capacitor Android project for mobile packaging

## Getting Started

### Prerequisites

- Node.js
- npm
- Supabase CLI, if you want to run or deploy the backend locally
- Android Studio, if you want to build the Android app

### Install Dependencies

```bash
npm install
```

### Run the Web App

```bash
npm run dev
```

The Vite dev server is configured to run on:

```text
http://localhost:3000
```

### Mobile Demo Mode

To open the mobile UI demo, add the demo query parameter:

```text
http://localhost:3000/?demo=mobile
```

### Build for Production

```bash
npm run build
```

The production build is written to the `build/` directory.

## Android Build Notes

The Capacitor config uses `build/` as the web output directory:

```json
{
  "appId": "com.example.myapp",
  "appName": "MyApp",
  "webDir": "build"
}
```

To refresh the Android project after a web build:

```bash
npm run build
npx cap sync android
```

Then open or build the Android project from Android Studio.

## Backend

The backend is implemented as a Supabase Edge Function using Hono. The deployed function folder is:

```text
supabase/functions/make-server-215f50be
```

The frontend builds the API URL from `src/utils/supabase/info.tsx`:

```ts
https://{projectId}.supabase.co/functions/v1/make-server-215f50be
```

Main backend capabilities include:

- User CRUD
- Job CRUD
- Course CRUD
- Application CRUD and status updates
- Analytics aggregation
- Recommendation and shortlist persistence endpoints

For more detail, see `src/BACKEND_DOCUMENTATION.md`.

## Important Notes

- Some authentication flows are demo-oriented and use browser `localStorage` for profile persistence.
- The app combines local/offline-friendly state with Supabase-backed API persistence.
- The `src/utils/supabase/info.tsx` file is marked as autogenerated and contains the current Supabase project ID and public anonymous key.
- The package metadata still uses the original generated project name, but the application and documentation refer to the project as ProMatchAI.
